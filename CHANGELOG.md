# 更新日志

本项目所有值得注意的变更都记录在这个文件里。

格式遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循[语义化版本](https://semver.org/lang/zh-CN/)（`主版本.次版本.修订号`：不兼容改动 / 新功能 / 修 bug）。

维护约定：
- 平时改代码，把变更随手记到 `[Unreleased]` 段
- 发版时把 `[Unreleased]` 改成版本号 + 日期，开新的空 `[Unreleased]`
- GitHub Release 的发布说明直接复制对应版本的段落

## [Unreleased]

### Fixed
- 内容搜索 `/api/content` 的本地 grep 回退会复用文本预览编码识别；GBK/ANSI 工程说明文件现在能被中文关键词搜到，命中片段也不会显示乱码。
- 文本编辑保存会沿用 `/api/read` 检测到的原始编码；GBK/ANSI 工程说明文件编辑后仍按 GBK 写回，不再悄悄转成 UTF-8，并新增真实 Chrome 保存回归覆盖用户点击保存路径。

### Tests
- 新增 GBK 文本内容搜索真实 API 回归，启动临时 Arca 服务并验证 `重要说明.txt` 可通过“参考设计”搜到且命中内容为可读中文。
- 文本预览新增真实工程文件回归，直接验证 `重要说明.txt` 这类 GBK/ANSI 老 Windows 说明文件通过 `/api/read` 和真实 Chrome 预览 UI 返回/显示可读中文且不含乱码。

## [1.0.93] - 2026-06-18

### Fixed
- 文件夹在预览窗格中不再只有“没有预览”的静态提示；占位预览会保留动作栏，可直接进入文件夹、打开属性、在系统文件管理器中显示或复制路径，减少从预览面板回到文件区/快捷键的往返。
- 删除 AI 对话前增加确认弹窗，取消时不会调用删除接口；确认删除当前对话后回到新对话，避免误点左侧记录里的删除按钮导致上下文丢失。
- AI 对话左侧会话列表的拖拽分隔条改为可见且更宽的命中区，拖动后会持久化宽度，长会话标题和路径更容易看清。
- “本地生成/整理成/导出可打印文档”这类承接前文并需要落盘的请求会强制走完整 Agent，而不是误走轻量快车道；Agent prompt 会继续带上本对话前文，减少同一会话里反问“生成什么文档”的情况。
- 普通 Word/docx 概念问答不再因为出现 `Word` 或 `docx` 字样就启用完整 Agent/Word 工具；只有审阅、批注、修订、规范检查或已有 `.docx` 附件时才走 Word 工具链，提升轻量问答速度。

## [1.0.91] - 2026-06-17

### Fixed
- 通用错误文案增加 `friendlyErrorText` 兜底，快捷方式目标、变更面板打开、预览复制、系统打开和复制失败等入口遇到空错误或裸对象时显示“未知错误/具体 error 字段”，不再把 `[object Object]` 或空尾巴暴露给用户。

- 任务模板卡片新增部门标签，跨部门搜索结果能直接看出模板属于通用、财务、采购、文控等哪个场景。

- 任务模板填空表单打开后会自动聚焦第一项输入框，支持 Enter 执行、Esc 返回模板列表并把焦点放回搜索框；和“开始执行”按钮共用同一条提交逻辑，空必填字段会在原输入框就地标红并设置 `aria-invalid`，输入后自动恢复，不会误发任务。
- 任务模板搜索框支持按 ArrowDown 直接聚焦第一张结果卡，结果卡之间可用 ArrowLeft/ArrowRight/ArrowUp/ArrowDown 移动，Esc 回到搜索框，搜到模板后可不离开键盘直接按 Enter 打开；按 Esc 清空后会在下一帧把焦点补回搜索框，避免浏览器 keydown 重建输入框后焦点丢失。
- 任务模板搜索输入时重绘结果不会再丢失输入焦点，连续输入“财务/合同/8D”等关键词时光标保持在搜索框内。
- 任务模板搜索词不再跨次打开残留；上次搜到空结果后，下次打开模板区会恢复默认列表，避免误以为模板库为空。
- AI 对话在当前非 Claude 服务商未配置 API key 时，会提前禁用发送按钮并打开设置提示，不再创建空失败回合。
- AI 设置页选择非 Claude 服务商但未填写 API key 时，会拦截保存并聚焦 key 输入框，不再提示保存成功却实际不可用。
- AI 对话和设置页会同步拦截自定义中转缺 Base URL、缺模型的状态，不再等后端报错才发现配置不可用。
- AI 设置页会随输入即时禁用/启用“保存并启用”，并在状态栏显示当前还缺 API key、Base URL 或模型。
- AI 对话会把 relay 断连、网络失败、API key 无效、限流/额度不足、模型不可用等原始错误归一成中文可操作提示，并在错误气泡里提供“重试”或“打开设置”，同时继续恢复输入草稿和附件。
- Agent 接管连续对话时会额外带入最近相关路径清单，从历史附件、工具调用和 AI 回复中提取并去重；工具输出的 `参考答案.docx` 这类相对路径会按会话目录还原为完整路径，并把 `xlsx 已生成`、`输出目录 已生成` 这类文件/目录路径后的说明文字截掉；用户说“刚生成的文件/打开它/修改它”时更容易直接落到真实文件。
- AI 回复里的路径胶囊新增目录路径识别，`D:\项目\输出目录`、`D:\New Project\Output Folder`、`"D:\New Project\Output Folder"`、`\\nas\文控\发布清单`、`file://server/share/folder` 这类没有扩展名的文件夹路径也能直接打开或复制。
- 文本预览 GBK/ANSI 解码增加真实 `/api/read` 回归脚本，临时生成 GBK `重要说明.txt` 并确认预览返回中文与 `encoding=gbk`。
- 文件区非选中 cursor 补齐透明边框，拖放提示在 drop、dragend、pointerup、pointercancel、mouseup、窗口失焦、切窗口可见性变化和 Esc 时统一清理，避免残留红框看起来像两个不同选中态。
- 文件区拖到文件卡片上的落点提示改为虚线透明背景，并覆盖 hover/focus 残影，不再和真实选中态共用“浅红底 + 红框”的视觉语言。
- 文件区非选中的拖放落点从红色虚线改为中性虚线，避免截图或短暂停留时看起来像“一个浅红选中 + 一个空心红框”的双选状态。
- 文件区强制重绘选中态时会先清掉 DOM 中所有旧 `.selected`，再按当前状态重画，避免状态栏显示“已选 1 项”但两个文件卡片都残留选中框。
- 文件区单击切换选中文件时也会强制清理旧 `.selected` DOM，避免状态栏只显示 1 项但上一个文件卡片仍残留红框。
- 文件区程序化选中新文件、AI 写完文件后自动定位等路径也会强制重绘选中态，并在 CSS 末尾兜底禁止非选中 hover/focus/cursor 画红框，避免看起来像两个文件处于不同选中状态。
- 最近文件/文件夹直接打开、搜索结果/最近列表的「打开所在位置」在父目录或目标目录访问失败时会显示权限、超时或目录不存在等具体原因，不再误报成“文件不在原位置”。
- 对话区复制回复、复制代码、复制表格和复制路径遇到桌面剪贴板桥或浏览器剪贴板异常时，会把具体原因显示在 toast 里，不再只提示笼统的“复制失败”。
- 内嵌终端或一键启动 Claude/Codex 失败时，会把 node-pty 返回的具体原因显示在 toast 里，不再只提示笼统的“终端启动失败”。
- 内嵌终端进程退出后按回车重开失败时，也会把 node-pty 返回的具体原因弹出提示，并清晰保留在 session 状态里，避免用户只看到一行终端红字。
- 终端启动/重开失败但底层没有返回原因时，终端内红字也会显示“未知错误”，和 toast/session 状态保持一致，不再留下空尾巴。
- 终端启动/重开失败提示 helper 对空 session 也统一兜底为“未知错误”，避免异常链路显示 `null/undefined`。
- 英文界面补齐“终端重开失败：...”动态错误翻译，避免语言切换后新增失败反馈仍残留中文。
- 右键「创建快捷方式」和「粘贴快捷方式」出现部分失败或全部失败时会显示最后一个后端错误原因，不再只提示笼统失败。
- 目录打开过程中遇到网络、权限或路径不存在等异常时会显示可理解的具体原因，不再只提示笼统的“打开失败”。
- Excel 预览的工作簿级/工作表级解析异常会转成“表格结构异常，建议用系统应用打开”，不再把 `Cannot read properties ... indexOf` 这类 JS 异常暴露给用户；已用真实 `PCB-S1073-00信号测试问题汇总V14 .xlsx` 回归 Sheet1/Sheet2/History。
- Word 预览解析异常会转成“文档结构异常，建议用系统应用打开”，不再把 docx/zip/OOXML 内部错误或 JS TypeError 直接暴露给用户；已用真实 `财务实习生考核_实操试题_参考答案.docx` 回归正常打开链路。
- 递归搜索 `/api/search` 失败时，文件区会显示失败页、保留关键词并提供“重试/返回当前目录”，不再只弹 toast 后停在“搜索中…”。
- 添加网络位置会先校验输入格式，空输入、`nas/share` 这类普通文本会直接提示示例并不触发后端访问；带空格/引号的 `file://server/share` 会先规范化再验证，指到文件时提示必须选择文件夹。
- 预览区选中文本「发到终端」的完成提示统一为“已发到终端，确认后再回车”，与 AI 回复代码块同类入口保持一致；同时补齐「发到对话」和选区发送反馈的英文词典，避免中英文界面文案漏同步。
- 预览面板「复制文件」和「在文件管理器中显示」的英文词典改用 system file manager/file manager，不再在 Windows 语境下显示 Finder。
- 英文界面里的“本机/全机”相关文案改用 this device / this computer，不再在 Windows 公司版里显示 This Mac。
- 内嵌终端不可用时的提示统一为“请使用桌面版或检查终端组件”，不再把所有失败都误说成“网页版没有终端”；预览片段发终端、Skills 调终端和底部终端切换共用同一提示。
