## 一、产品定名(已决策,2026-06-12)

**灵匣 / Arca**——"文件夹操作 + Agent 结合体"的字面翻译:
- **匣** = 文件的容身处(文件匣)——文件管理这半边
- **灵** = 匣中有灵,会自己整理、替你办事——Agent 这半边
- 叙事:上游叫"翻箱"(人翻箱子),我们叫"灵匣"(匣子自己会干活),同宗进化
- 命名权重(用户定):6 分功能、2 分公司品牌(信步 SEAVO)、2 分文化底蕴
- **英文名 Arca**(2026-06-12 定,用户否决拼音 LingXia):拉丁语"匣/柜",与 Ark(约柜,"匣中有灵"的西方原型)同源;品牌区写法 SEAVO ARCA
- 可用标语:匣中有灵 / 文件匣,自己会干活
- 否决过的方向(别再提):拼音直转 LingXia、南桥/板桥(行业件名)、板载/点亮/常亮(黑话)、信手/闲庭/则灵(套用品牌文脉)、干将/搭手/办妥(帮手拟人)、总装/百工台/工坞(泛工作台)
- 候选过的英文名:Emberbox/Sparkcase/Tinderbox(用户选了 Arca)

13. **AI 双通道上下文桥接**(2026-06-16 Windows):修复同一会话里前一轮走快车道、后一轮因“生成 Word/写文件”切到完整 Agent 时上下文断裂的问题。Agent 首次接管轻量会话会收到最近前文，并把“上面/刚才/这个/生成文档”等指代解析为最近一轮助手产出，避免反问“生成什么文档”。
14. **AI Agent 预热池**(2026-06-16 Windows):确认慢点不是 SDK import(约 0.1s),而是 Claude Code 引擎子进程初始化(约 10-12s)。直聊或 Agent 回合后后台调用 SDK `startup()` 预启动下一次 Agent,下次文件/目录任务优先复用 warm query。真实 DeepSeek 写文件 E2E:无预热 `system/init` 10.34s、首字 13.04s、完成 13.18s；预热命中后 `system/init` 0.24s、首字 3.63s、完成 3.80s,文件实际落盘。
15. **AI 错误提示友好化**(2026-06-17 Windows):对话面板把 relay/网络/API key/限流/模型不可用等原始上游错误转成中文可操作提示,不再把 `relay: fetch failed ECONNREFUSED`、`invalid api key`、`Direct API 429` 直接甩给用户;错误气泡会给“重试”或“打开设置”动作,失败时仍恢复输入草稿和路径附件。

## 五、发版流程(v1.0.0 已于 2026-06-12 走通)

- **私库**:https://github.com/kidyear/arca(private,main=company 分支)
- **发新版**:改 package.json version → commit → `git tag v新版本 && git push seavo main --tags` → Actions 自动构建并发 Release(workflow 已配 contents:write)
- 若 tag push 没触发(新仓库偶发),手动 dispatch:`POST /repos/kidyear/arca/actions/workflows/build-windows.yml/dispatches {"ref":"v标签"}`
- **内网更新源**:公司内网服务器(Windows Server + IIS),发布目录放 Arca-Setup-X.Y.Z.exe + latest.json({"version","url"});客户端默认源由 CI 从仓库变量 COMPANY_UPDATE_URL 注入(electron/company.json,gitignored),不写进源码。实际地址见 GitHub 仓库 Settings→Variables
- **踩过的坑**:nsis 块不认 writeUpdateInfo(dmg 专属);新仓库 GITHUB_TOKEN 默认只读要加 permissions: contents:write;GitHub OAuth 推 workflow 文件需要 workflow scope(设备流授权)
- **v1.0.2 真机大坑(asar spawn)**:SDK 引擎二进制(claude.exe,平台包 claude-agent-sdk-win32-x64)被 SDK 从 app.asar 逻辑路径启动——fs 读有 Electron asar 透明转发所以"exists",spawn 没有转发所以"failed to launch"。修法:ai.js engineBinaryPath() 解析真实 unpacked 路径显式传 pathToClaudeCodeExecutable;asarUnpack 必须含 claude-agent-sdk-*(平台包,嵌套/平铺两种布局)。Anthropic 兼容端点多不实现 /v1/models→模型列表走各家 OpenAI 风格 modelsUrl 回退。NSIS 升级安装免卸载,配置在 ~/.fanbox 与安装目录无关
- **v1.0.5 真机大坑(全局 Claude 设置污染 relay)**:用户机器的 `~/.claude/settings.json` 可能配置了 `env.ANTHROPIC_BASE_URL`/`ANTHROPIC_AUTH_TOKEN`(本机中转、代理、其他模型等)。SDK 默认读取 user/project/local settings，优先级会覆盖 Arca 传入的 relay 环境变量，表现为仍然 `ConnectionRefused` 且 `~/.fanbox/engine.log` 没有 `relay 上游失败`。修法:ai.js 的 query options 加 `settingSources: []`，嵌入式应用以 Arca 的 Provider 配置为准。
